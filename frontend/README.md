# Frontend — мини-апп «Курьер — доставка на маркетплейс»

Фронтенд на основе текущего прототипа: тот же UI, стили вынесены в CSS, логика — в ES-модули. Сборка через Vite.

## Структура

- `index.html` — разметка всех экранов (как в корневом прототипе)
- `src/styles/main.css` — все стили приложения
- `src/main.js` — точка входа, инициализация и запуск авторизации
- `src/api.js` — работа с API (getApiBase, getToken, api(), 401 → повторный вход)
- `src/auth.js` — авторизация Telegram, привязка телефона, выбор роли
- `src/app.js` — навигация (showScreen), загрузка данных (заявки, профили, карта), привязка кнопок и форм

## Запуск

```bash
cd frontend
npm install
npm run dev
```

Откроется http://localhost:5173. Прокси на бэкенд: запросы к `/api` уходят на http://localhost:3000.

## Сборка

```bash
npm run build
```

Результат в папке `dist/`. Для публикации мини-аппа залейте содержимое `dist/` на хостинг и укажите URL в настройках бота Telegram.

## Настройка API

По умолчанию в браузере без Telegram используется `http://localhost:3000/api`. В Mini App — текущий origin + `/api`. Чтобы задать свой URL, перед загрузкой страницы выполните:

```html
<script>window.APP_API_BASE = 'https://ваш-сервер.com/api';</script>
```
